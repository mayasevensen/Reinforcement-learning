\subsection*{Exercise 1}
\begin{enumerate}
    \item (*) When using a linear functions for feature engineering you transform the original state into features. This creates a new feature vector from the raw input. The purpose is to represent the state in a more useful way for the model.
    For linear approximation on the other hand the goal is to approximate the target function.
    \[f(s;w) = w^T\phi(s)\]
    The parameters $w$ are learned from data. It maps features to predictions.
   
    \item Let the approximate value function be $\hat{v}(s;w)=w^\top \phi(s)$
    The TD($n$) target is
    \[ G_t^{(n)} = R_{t+1} + \gamma R_{t+2} + \dots + \gamma^{n-1}R_{t+n} + \gamma^n \hat{v}(S_{t+n};w)\]
    The semi-gradient update for the weights is
    \[w \leftarrow w + \alpha \big(G_t^{(n)} - \hat{v}(S_t;w)\big)\nabla_w \hat{v}(S_t;w)\]

    For linear approximation $\nabla_w \hat{v}(S_t;w)=\phi(S_t)$, giving
    \[w \leftarrow w + \alpha \big(G_t^{(n)} - \hat{v}(S_t;w)\big)\phi(S_t)\]

    \item (*) The policy improvement theorem assumes access to the exact action-value function $q_\pi(s,a)$. With function approximation we only have an estimate $\hat{q}(s,a)$, which may contain errors. A policy that is greedy with respect to $\hat{q}$ may therefore choose actions that are not actually better. Consequently, the step in the proof assuming $q_\pi(s,\pi'(s)) \geq q_\pi(s,\pi(s))$ no longer holds, and policy improvement is not guaranteed.

    \item (*) Replacing the tabular action-value function with a neural network introduces the challenge of combining function approximation, bootstrapping, and off-policy learning. This combination is known to cause instability in reinforcement learning. 

    In the tabular case, each update affects only a single state–action value, which guarantees stable learning. With a neural network, however, updates affect many states simultaneously due to shared parameters. When combined with TD($\lambda$) bootstrapping and off-policy data, errors can propagate through the value estimates and may lead to unstable learning or even divergence. 
    
\end{enumerate}

\subsection*{Exercise 3}
\begin{enumerate}
    \item (*) Monte Carlo Tree Search (MCTS) can improve decision making by simulating many possible future trajectories and selecting actions based on their expected outcomes. However, several considerations are necessary before applying it to a self-driving car problem.

    First, MCTS requires a model or simulator of the environment in order to generate simulated trajectories. The quality of the resulting policy therefore depends on the accuracy of this model. Second, MCTS can be computationally expensive because it requires many simulations for each decision. In a self-driving scenario, decisions need to be taken quickly in real time, which may limit the number of simulations that can be performed.
    
    Finally, driving environments typically have continuous state and action spaces, while MCTS is more naturally suited to discrete decision problems. For these reasons, the use of MCTS may be beneficial only if the environment can be simulated efficiently and the decision space can be suitably discretized.
\end{enumerate}
