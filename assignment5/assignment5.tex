\item
\begin{textbf}
    yes, we can define an optimal deterministic policy for this MDP. The optimal policy would be:
    \begin{enumerate}
        \item In state $s_0$, take action \text{prep} to move to state $s_1$.
        \item  In state $s_1$, take action \text{sugar} to move to state $s_2$.
        \item In state $s_2$, take action \text{cook} to move to state $s_6$ and receive a reward of +10.
    \end{enumerate}
    This policy is optimal because it leads to the highest possible reward of +10 by reaching state $s_6$.
    Any deviation from this policy would either lead to a lower reward (e.g., going to state $s_7$ with a reward of -10), a delayed reward (like taking action \text{taste} in $s_2$ leading to state $s_4$), or no reward at all (e.g., waiting in state $s_0$ or taking the restart action in states $s_2$ or $s_3$). Therefore, the optimal deterministic policy is to always choose the actions that lead to state $s_6$.
    This policy is simply the shortest path to the highest reward.
\end{textbf}

\item
\begin{textbf}
    Yes, an optimal stochastic policy exists. 
    However, since the environment is fully deterministic and there is a unique optimal trajectory leading to reward +10,
    any randomness that assigns non-zero probability to suboptimal actions (e.g., adding salt) will reduce the expected return.
Thus, the optimal stochastic policy is actually just deterministic, meaning the agent always chooses the best action with probability 1 and never chooses any other actions.
    \[
\pi(a \mid s) =
\begin{cases}
1 & \text{if } a = a^*(s), \\
0 & \text{otherwise}.
\end{cases}
\]

In particular, the agent always chooses actions that lead to the sugary cookies and avoids all actions leading to the salty outcome.
\end{textbf}

\item 
No, an optimal deterministic policy does not exist.
The agent cannot observe the latent variable $\sigma \in \{\text{sugar}, \text{salt}\}$, so in the enriched bowl state it cannot distinguish between the two underlying cases. A deterministic policy must choose the same action regardless of $\sigma$.
However, the optimal behavior depends on information: the agent should first \emph{taste} to reveal $\sigma$, and then choose \emph{cook} if $\sigma=\text{sugar}$ or \emph{restart} if $\sigma=\text{salt}$. This requires conditioning actions on information that is not available in the current observation.
Thus, no deterministic memoryless policy can achieve optimal performance. An optimal policy must either use memory (history-dependent) or be stochastic.

\item
Yes, an optimal stochastic policy exists.

Due to partial observability, the agent must act under uncertainty in the enriched bowl state. The optimal strategy is to first gather information and then act optimally based on that information.

An optimal policy is:
\[
\pi(\text{taste} \mid \text{enriched bowl}) = 1,
\]
and then
\[
\pi(\text{cook} \mid \text{sugar sampling}) = 1, \quad
\pi(\text{return} \mid \text{salt sampling}) = 1.
\]

After returning to the enriched bowl from salt sampling, the agent again follows:
\[
\pi(\text{taste} \mid \text{enriched bowl}) = 1.
\]

Thus, the agent repeatedly tastes until it identifies sugar, and only then chooses to cook, ensuring the maximal expected reward.

\item
The key difference between the original MDP and the new MDP is that the new MDP introduces partial observability through the latent variable $\sigma$ that determines whether the ingredient added is sugar or salt. 
In the original MDP, the agent has full observability and can directly choose actions based on the current state.
In contrast, in the new MDP, the agent cannot directly observe $\sigma$ and must rely on actions (like tasting) to gain information about it.
This means that an optimal policy may require memory or stochasticity, as the agent needs to condition its actions based on the information gathered.

\item
Yes, a natural modification to the original MDP is to make the latent variable $\sigma$ observable, but only after taking a specific action (e.g., tasting). 
This creates a scenario where the agent must first take an exploratory action to gain information about the state of the environment before it can make an informed decision.
This would restore full observability while still requiring the agent to gather information. 
Alternativly we can augment the state space to incude the history of actions take, allowing the agent to infer the latent variable from past actions and observations. 
In both cases we remove partial observability and simplify the policy design.
